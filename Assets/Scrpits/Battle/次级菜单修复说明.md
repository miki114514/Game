# 次级菜单修复完整说明

## 🔧 已修复的问题

### 问题1: 次级菜单只显示箭头，没有UI

**根本原因**: itemPrefab 为空或 content 未配置 Layout Group  
**修复内容**:

- ✅ 在 ShowSubCommands() 中添加了 null 检查和详细错误日志
- ✅ 改用 anchoredPosition 代替 localPosition 确保在 Canvas 坐标系正确显示
- ✅ 添加了调试信息方便诊断

### 问题2: W/S 切换时原指令面板也跟着变

**根本原因**: CommandMenuUi 和 SubCommandPanelUi 都在同时处理键盘输入  
**修复内容**:

- ✅ CommandMenuUi.Update() 当 `inSubMenu = true` 时完全跳过输入处理和UI更新
- ✅ 只由 SubCommandPanelUi 在 isActive = true 时处理输入
- ✅ 去除了 CommandMenuUi 中的 HandleSubMenuInput() 方法
- ✅ 简化了 UpdateArrowPosition() 逻辑

### 问题3: 菜单关闭逻辑混乱

**修复内容**:

- ✅ 添加了 CommandMenuUi.RequestCloseSubMenu() 公开方法
- ✅ SubCommandPanelUi 的 Escape 和 选择确认 都通过此方法关闭菜单
- ✅ 确保菜单状态同步和清晰的调用链

## ⚙️ 必要的Inspector配置

### 1️⃣ SubCommandPanelUi 组件（在主菜单 Panel 上查找）

```
SubCommandPanelUi 脚本
├─ itemPrefab (GameObject) ⭐ 关键
│  └─ 需要这个结构:
│     ├─ Base (Image)
│     │  └─ Text (TextMeshProUGUI)
│     └─ Select (Image)
│        └─ Text (TextMeshProUGUI)
│
├─ content (RectTransform) ⭐ 关键
│  └─ ⚠️ 必须添加 VerticalLayoutGroup 组件!
│
├─ arrow (RectTransform)
│  └─ 箭头图片
│
├─ arrowOffset
│  └─ 设置合适的偏移量 (默认: -50, 0)
│
└─ battleManager (BattleManager)
   └─ 自动关联
```

### 2️⃣ content 的 RectTransform 必须配置

**关键步骤:**

1. 选中 Content 节点
2. 添加组件 → VerticalLayoutGroup
3. 设置以下参数:
   - Preferred Height: ✓ (勾选)
   - Child Force Expand Height: ✓ (勾选)
   - Child Force Size Height: ✓ (勾选)

### 3️⃣ itemPrefab 预制体的完整设置

```yaml
ItemPrefab:
  name: "SubCommandItem"

  Base (Image 组件):
    - Color: 设置常态颜色 (如白色)
    - RaycastTarget: false
    └─ Text (TextMeshProUGUI):
       - Color: 黑色或你的文字颜色
       - Font Size: 36

  Select (Image 组件):
    - Color: 设置选中颜色 (如高亮色)
    - RaycastTarget: false
    └─ Text (TextMeshProUGUI):
       - Color: 黄色或对比颜色
       - Font Size: 36

  LayoutElement (必须添加):
    - Preferred Height: 60
```

**注意**: 不要启用任何 Button 或 Selectable 组件，这会干扰输入处理！

## 📋 代码改动摘要

### CommandMenuUi.cs

```csharp
// 改动1: Update() 方法 - 当 inSubMenu 时完全禁用输入
void Update()
{
    if (!inSubMenu)  // 只有在非子菜单时才处理
    {
        HandleMainMenuInput();
        UpdateUI();
        UpdateArrowPosition();
    }
}

// 改动2: 新增公开方法供 SubCommandPanelUi 调用
public void RequestCloseSubMenu()
{
    CloseSubMenu();
}

// 改动3: OpenSubMenu() 添加了激活标志
if (subCommandPanelUi != null)
{
    subCommandPanelUi.commandMenuUi = this;
    subCommandPanelUi.isActive = true;  // ← 新增
    subCommandPanelUi.ShowSubCommands(subCommands, cmd);
}
```

```csharp

public void ShowSubCommands(List<SubCommand> commands)
```

## 🐛 如果仍然不显示:

1. **Console 错误检查**
   - 查看是否有 "[SubCommandPanelUi] itemPrefab 未配置" 的错误
   - 查看是否有 "Content 未配置" 的错误

2. **检查 itemPrefab**

   ```
   Window → TextMeshPro → Import TMP Essential Resources
   (如果有字体缺失问题)
   ```

3. **检查 content 的 RectTransform**
   - 选中 content
   - Inspector → RectTransform → Width/Height 不为 0
   - 应该看到 VerticalLayoutGroup 组件

4. **检查 Hierarchy 中的预制体实例**
   - 运行游戏
   - 选择道具后查看 Hierarchy
   - 应该在 SubCommandPanel 下看到生成的 Item 对象
